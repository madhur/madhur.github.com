Convert claims based login name in SharePoint Client object model

29 January 2014
Tweet

Recently, I was working on a silverlight solution and came across an issue where People WebService returned me the duplicate results for the people who are already part of the site collection. This issue can occur if SharePoint is using claim based authentication model.

On close inspection I found that the returned duplicate results were having different login names. For example if one result was ads\john then the other one was i:0#.w|ads\john.
